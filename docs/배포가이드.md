# 배포 가이드

## 1. 개발 환경 실행

```bash
npm start          # Electron 데스크톱 앱으로 실행
python server.py   # 웹 브라우저 모드로 실행 (폴백)
```

## 2. 빌드

```bash
npm run build      # dist/ 폴더에 인스톨러 생성 (로컬 배포용)
```

결과: `dist/Scoresentation Setup X.X.X.exe`

## 3. 업데이트 배포

### 사전 준비 (최초 1회)
1. GitHub > Settings > Developer settings > Personal access tokens > Tokens (classic)
2. `repo` 권한으로 토큰 생성
3. 토큰을 안전한 곳에 보관

### 배포 절차

```bash
# 1. 코드 수정 완료 후 package.json 버전 올리기
#    "version": "1.0.0" → "1.0.1"

# 2. 커밋 & 푸시
git add -A
git commit -m "v1.0.1 ..."
git push

# 3. 토큰 설정 (Windows)
set GH_TOKEN=ghp_xxxxxxxxxxxx

# 4. 빌드 + GitHub Release 생성
npm run publish
```

`npm run publish` 완료 후 GitHub Releases에 자동 생성되는 파일:
- `Scoresentation Setup X.X.X.exe` (인스톨러)
- `latest.yml` (버전 메타데이터)

### 사용자 측 자동 업데이트 흐름
1. 앱 실행 시 GitHub Releases에서 `latest.yml` 확인
2. 새 버전 발견 → 백그라운드 다운로드
3. 다운로드 완료 → "새 버전이 다운로드되었습니다. 재시작하시겠습니까?" 알림
4. "재시작" 클릭 → 앱 종료 후 자동 설치 및 재실행

### 버전 규칙
- `X.Y.Z` (semver)
- 패치 수정: `1.0.0` → `1.0.1`
- 기능 추가: `1.0.0` → `1.1.0`
- 대규모 변경: `1.0.0` → `2.0.0`
- `package.json`의 `version` 필드가 릴리스 버전이 됨

## 4. 프로젝트 구조

```
scoresentation/
├── main.js              # Electron 메인 프로세스
├── preload.cjs          # Context bridge (IPC)
├── package.json         # 의존성 + 빌드 설정
├── main/                # 백엔드
│   ├── db.js            # SQLite 리포지토리 (better-sqlite3)
│   └── media.js         # 미디어/이미지 폴더 관리
├── src/                 # 프론트엔드 소스
│   ├── index.html       # 프레젠테이션 진입점
│   ├── present.js/.css  # 프레젠테이션 로직/스타일
│   ├── editor.html      # 편집기 진입점
│   ├── editor.js/.css   # 편집기 로직/스타일
│   ├── notes.js/.css    # SVG 악보 렌더링 엔진
│   ├── presentation.js/.css # 슬라이드 기본 스타일
│   ├── storage.js       # 곡 저장소 (Electron IPC / fetch 듀얼)
│   └── setlistStorage.js # 셋리스트 저장소 (동일 듀얼)
├── fonts/               # Bravura, Freesentation 폰트
├── data/                # DB + 미디어 + 이미지
├── docs/                # 개발 문서
├── tools/               # NWC 변환 도구 + 웹 폴백 서버
├── dist/                # 빌드 결과물 (gitignore)
└── node_modules/        # npm 패키지 (gitignore)
```

## 5. 참고

- 코드 서명 없이 배포 시 Windows SmartScreen 경고가 표시될 수 있음
- `data/` 디렉터리는 패키징 시 `extraResources`로 asar 밖에 배치 (SQLite 쓰기 가능)
- 개발 모드(`npm start`)에서는 자동 업데이트를 건너뜀 (`app.isPackaged` 체크)
