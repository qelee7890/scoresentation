# 세션시작프롬프트

## 표준 버전
```text
작업 시작 전에 프로젝트 문서를 먼저 읽고 현재 상태를 파악해.
읽기 순서:
1. docs/현재동작명세.md
2. docs/개발일지.md
3. docs/개발기획서.md

읽은 뒤에는 아래 3가지를 짧게 정리해:
1. 현재 실제 동작
2. 최근 변경 사항
3. 이번 작업에 바로 필요한 맥락

그 다음 실제 작업을 진행해.
작업 중 현재 동작 규칙이 바뀌면 docs/현재동작명세.md를 갱신하고,
세션 작업 내용과 결정 사항은 docs/개발일지.md에 기록해.
docs/개발기획서.md는 제품 목표, 범위, 장기 방향이 바뀌는 경우에만 수정해.
기획서에는 구현 로그나 세션 기록을 적지 마.
```

## 실전형 짧은 버전
```text
문서부터 읽고 이어서 작업해.
순서: docs/현재동작명세.md -> docs/개발일지.md -> docs/개발기획서.md
읽은 뒤 현재 동작 / 최근 변경 / 이번 작업 맥락을 짧게 요약하고 작업해.
작업 후에는 docs/개발일지.md와 docs/현재동작명세.md를 갱신해.
docs/개발기획서.md는 기획 변경이 있을 때만 수정해.
```

## 용도
- 새 대화 시작 시 그대로 붙여넣기
- 다른 작업을 이어받을 때 현재 문서 체계를 유지하기 위한 공통 시작 문구
