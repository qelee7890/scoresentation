# 현재동작명세

> 이 문서는 현재 저장소에서 실제로 동작하는 편집기/프레젠테이션 규칙을 정리한 기준 문서다.
> 초기 목표와 제품 방향은 `개발기획서.md`, 세션별 변경 이력은 `개발일지.md`를 기준으로 한다.

## 1. 문서 목적
- 현재 구현된 동작을 빠르게 확인하기 위한 운영 문서
- 다음 세션에서 “지금 무엇이 실제로 동작하는가”를 바로 파악하기 위한 기준 문서
- 기획 문서와 작업 로그 사이의 빈틈을 메우는 역할

## 2. 현재 앱 구조
- 프레젠테이션 진입점: `index.html`, `app.js`
- 편집기 진입점: `editor.html`, `editor.js`, `editor.css`
- 샘플 진입점: `sample-flowers.html`
- 악보 렌더링 엔진: `notes.js`, `notes.css`
- 저장소 계층: `storage.js`
- SQLite 서버: `server.py`
- 곡 데이터 DB: `data/scoresentation.db` (primary source, 559곡)
- NWC 변환 도구: `nwc_bridge.mjs`, `nwc_to_hymns.py`, `nwc_batch_convert.py`
- `hymns.json`, `ccm.json`은 삭제됨 (DB로 완전 이전, gitignore 처리)

## 3. 데이터 기준

### 3.1 곡 데이터
- 한 곡 데이터는 최소 `id`, `category`, `title`, `key`, `timeSignature`, `verses`, `chorus`를 가진다.
- 찬송가 데이터는 기존 호환을 위해 `number`, `newNumber`를 계속 가질 수 있다.
- `verses[n].korean`, `verses[n].english`는 슬라이드 줄 배열이다.
- `verses[n].notes`, `chorus.notes`는 슬라이드별 악보 데이터 배열이다.

### 3.2 notes 구조
```json
{
  "0": [
    { "pitch": "G4", "duration": "8" },
    null,
    { "pitch": "A4", "duration": "q", "beamGroup": 1 }
  ],
  "1": [
    { "pitch": "C5", "duration": "8." }
  ]
}
```

- 객체 키 `"0"`, `"1"`은 가사 줄 인덱스다.
- 배열 인덱스는 글자 인덱스 기준이다.
- 값은 `null` 또는 음표 객체다.
- 음표 객체는 현재 `pitch`, `duration`, 선택적으로 `beamGroup`를 가진다.

### 3.3 pitch 표기 체계
- 현재 pitch 라벨은 예전 기준보다 한 단계 낮춰 저장한다.
- 예:
  - `D5 -> C5`
  - `C5 -> B4`
  - `B5 -> A5`
- 이 규칙은 렌더링 높이를 바꾸는 것이 아니라, 같은 높이에 붙는 pitch 이름을 바꾸는 것이다.
- 기존 데이터는 `pitchLabelVersion = 2` 마이그레이션으로 한 번만 변환된다.

### 3.4 duration 표기
- `w`: 온음표
- `h`: 2분음표
- `q`: 4분음표
- `8`: 8분음표
- `16`: 16분음표
- 점음표는 접미사 `.` 사용
  - 예: `q.`, `h.`, `8.`

## 4. 프레젠테이션 동작
- `server.py`를 통해 SQLite DB(`data/scoresentation.db`)에서 전체 곡 목록을 로드한다.
- DB에 559곡(찬송가 558 + CCM 1)이 저장되어 있으며, 555곡에 NWC 멜로디 notes가 포함되어 있다.
- `notes` 데이터가 있는 곡은 프레젠테이션에서 악보를 함께 렌더링한다.
- `?song=46` 같은 쿼리로 곡을 직접 열 수 있다.
- 기존 `?hymn=46` 쿼리도 하위 호환으로 계속 읽는다.
- `server.py` 없이 열면 곡 데이터를 로드할 수 없다.

## 5. 편집기 동작

### 5.1 기본 원칙
- 편집기는 기본적으로 “가사 기준 악보 편집기”다.
- 텍스트를 수정하면 오선지 길이와 음표 위치를 즉시 다시 계산한다.
- 슬라이드 진입, 폰트 로드 완료, 리사이즈 시 레이아웃을 다시 계산한다.
- 좌측 검색 패널에서 hymn/ccm 전체를 제목 또는 모든 슬라이드 가사로 검색할 수 있고, `Enter` 또는 검색 버튼으로 실행하며 결과는 최대 10개까지 표시한다.

### 5.2 가사 편집
- 한글/영문 줄은 `contenteditable`로 편집한다.
- 가사를 수정하면:
  - 현재 줄의 문자 위치를 다시 측정
  - 오선지 길이를 다시 계산
  - 음표 `x` 위치를 새 글자 중심 기준으로 다시 렌더링
- 한글/영문 줄에서 `Enter`:
  - 현재 줄을 두 줄로 나눈다.
- 한글/영문 줄에서 `Ctrl + Enter`:
  - 현재 커서 이후 텍스트를 같은 절 또는 후렴 안의 새 슬라이드로 보낸다.
- 줄 시작에서 `Backspace`, 줄 끝에서 `Delete`:
  - 인접한 두 줄을 한 줄로 합친다.
- 한글 줄 분할/병합 시 해당 줄의 `notes` 라인 인덱스도 함께 분할/병합된다.
- `Ctrl + Enter`로 슬라이드를 나눌 때도 한글 기준 `notes`가 함께 다음 슬라이드로 이동한다.
- export/save 시 수정된 가사도 같이 포함된다.

### 5.2.1 후렴 마커
- 절 슬라이드 가사에 `<후렴>` 마커가 들어가면 해당 슬라이드는 후렴 슬라이드로 옮긴다.
- 마커는 분류용으로만 사용하고, 실제 저장 텍스트에서는 제거된다.
- 기존 `chorus` 데이터에 있는 슬라이드는 계속 후렴으로 유지된다.

### 5.2.2 슬라이드 추가/삭제
- 편집기 툴바에서 현재 슬라이드 뒤에 새 슬라이드를 추가할 수 있다.
- 현재 슬라이드를 삭제할 수 있다.
- 내용이 있는 슬라이드를 삭제하면 같은 절/후렴의 직전 슬라이드 뒤에 내용을 이어 붙인 뒤 삭제한다.
- 내용이 있는 첫 슬라이드는 앞 슬라이드가 없으므로 바로 삭제하지 않는다.
- 마지막 남은 단일 슬라이드는 삭제하지 않는다.
- 절 슬라이드와 후렴 슬라이드 모두 같은 방식으로 다룬다.

### 5.3 음표 생성
- 빈 오선지 위치에서만 회색 프리뷰 음표가 보인다.
- 빈 위치 클릭으로 새 음표를 만든다.
- 새 음표 pitch 판정은 오선지 기준 높이로 계산한다.

### 5.4 기존 음표 편집
- 기존 음표 위에서는 새 음표 프리뷰를 띄우지 않는다.
- 기존 음표 클릭:
  - 인라인 메뉴를 연다.
- 기존 음표 드래그:
  - 음정(pitch)을 변경한다.
- 기존 음표 더블클릭:
  - 음표를 삭제한다.

### 5.5 기존 음표 인라인 메뉴
- 현재 메뉴 동작:
  - `길이 변경`
  - `점 추가` 또는 `점 제거`
  - `삭제`
- `길이 변경` 클릭 후 후속 박스에서 선택 가능:
  - 8분
  - 4분
  - 2분
  - 1분
- 예전 `D` 키 점음표 모드는 제거되었다.

### 5.6 조표 편집
- 음자리표 오른쪽 조표 영역에서 편집한다.
- 일반 클릭:
  - 플랫 기준 조표 추가
- `Ctrl` + 클릭:
  - 샵 기준 조표 추가
- 더블클릭:
  - 마지막 조표 제거
- 조표 변경 시 함께 갱신되는 항목:
  - `hymn.key`
  - 모든 슬라이드의 `slide.key`
  - 악보 레이아웃
  - 화면 상단 찬송가 메타 정보
  - 현재 슬라이드 정보 줄의 `음조 · 박자`

## 6. 좌표 및 히트테스트 규칙

### 6.1 시각 좌표와 pitch 좌표 분리
- 선택 원, 음표 메뉴 위치, 음표 헤드 히트테스트는 보이는 헤드 중심 기준을 사용한다.
- pitch 계산은 오선지 기준 높이를 사용한다.
- 이 둘은 의도적으로 분리되어 있다.

### 6.2 드래그 판정
- 기존 음표 드래그 시작은 실제 음표 헤드 근처에서만 잡힌다.
- 빈 오선지 드래그는 다중 선택 박스로 처리된다.

### 6.3 드래그 선택의 x 좌표
- 연결선 드래그 박스의 `x` 판정은 `lineEl._layout.positions[charIndex]` 기준이다.
- SVG 내부 마진 좌표와 섞지 않는다.

## 7. 보조선 규칙
- 현재 보조선 규칙은 새 pitch 표기 체계에 맞춰 조정되어 있다.
- `G5`, `B5` 위쪽에는 불필요한 보조선이 없어야 한다.
- `C4`는 중앙 보조선이 표시되어야 한다.
- 하단(`C4` 이하) 보조선은 반 칸 아래로 보정한다.
- 상단(`A5` 이상) 보조선은 한 칸 위로 보정한다.

## 8. 연결선 규칙

### 8.1 선택 진입
- 빈 오선지 드래그로 여러 음표를 선택할 수 있다.
- `Ctrl` 또는 `Shift` + 기존 음표 클릭으로 선택을 추가할 수 있다.
- 별도 beam 모드 진입은 사용하지 않는다.

### 8.2 선택 조건
- 연결선 선택은 기존 8분음표, 16분음표에서만 가능하다.
- 같은 줄에서만 유지된다.
- 같은 기본 길이끼리만 가능하다.
  - 가능: `8 + 8`, `16 + 16`
  - 불가: `8 + 16`
- 선택 구간 사이에 다른 음표가 끼어 있으면 선택 자체를 막는다.
- 이 경우 메시지:
  - `인접하지 않은 음표들이 선택되었습니다.`

### 8.3 적용 방식
- 선택 후 인라인 메뉴에서:
  - `연결선 적용`
  - `연결선 해제`
  - `선택 취소`
- 잘못된 연결선 후보는 적용 단계가 아니라 선택 단계에서 바로 막는다.
- orphan `beamGroup`은 길이 변경/삭제/텍스트 길이 조정 뒤 정리한다.

## 9. 저장 및 불러오기
- DB(`data/scoresentation.db`)가 유일한 곡 데이터 소스다.
- `server.py`로 실행하면 `/api/hymns`를 통해 SQLite DB에서 전체 곡을 로드하고 저장/삭제한다.
- 프레젠테이션과 편집기 모두 DB에서 곡 목록을 빌드한다 (`buildHymnMapFromStorage`).
- `hymns.json`은 삭제되었고 앱에서 사용하지 않는다.
- 백엔드 API를 사용할 수 없으면 `storage.js`가 브라우저 `localStorage`로 자동 폴백한다.
- 저장본은 곡 `id` 기준으로 저장한다.
- 편집기에서:
  - 현재 곡 저장
  - 저장본 삭제
  - JSON import
  - JSON copy/download
  - 하단 JSON을 직접 수정하면 유효한 형식일 때 현재 슬라이드와 곡 데이터에 즉시 반영

### 9.1 최근 반영 사항
- 좌측 곡 검색은 실시간 필터링이 아니라 사용자가 `Enter`를 누르거나 검색 버튼을 눌렀을 때만 실행된다.
- 검색 대상은 hymn/ccm 전체 곡의 제목과 모든 슬라이드 가사이며, 결과는 최대 10개까지 표시된다.
- 하단 JSON 편집은 입력 중 짧은 지연 후 현재 곡과 슬라이드에 반영된다.
- 하단 JSON 편집은 포커스를 벗어날 때 다시 검증하고, 유효하면 정리된 포맷으로 다시 출력된다.
- 하단 JSON 편집으로 반영된 변경은 Undo/Redo 히스토리에도 한 번의 편집으로 기록된다.

## 10. NWC 변환 파이프라인
- `nwc_bridge.mjs`: Node.js에서 zz85/nwc-viewer 파서로 NWC 파일을 파싱, 멜로디 JSON 출력
- `nwc_to_hymns.py`: 멜로디를 가사 슬라이드/줄에 1:1 매핑하여 notes 필드 생성
- `nwc_batch_convert.py`: 557개 NWC를 일괄 변환하여 DB에 저장 (555곡 성공)
- NWC pitch는 표준 음이름(A4, B4 등)으로 추출, v2 pitch 변환은 적용하지 않음
- NWC V170 NoteCM 구조: `data1(12B) + count(2B) + children(count×8B)`, 마지막 child가 소프라노
- nwc-viewer 파서는 `lib/nwc2xml/`에 프로젝트 내부 포함 (zz85/nwc-viewer v2.3 기반)
- `package.json`은 프로젝트 메타 정보만 포함, npm 외부 의존성 없음
