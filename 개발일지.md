# 개발일지

## 프로젝트 현재 상태
- 저장소는 정적 HTML/CSS/Vanilla JS 기반 프론트와 Python SQLite 서버를 함께 둘 수 있는 찬송가 프레젠테이션/편집기 프로토타입이다.
- 메인 재생 화면은 `index.html`, `app.js`, `presentation.js`, `notes.js`를 사용한다.
- 편집 화면은 `editor.html`, `editor.js`, `editor.css`, `notes.js`를 사용한다.
- 저장 계층은 `storage.js`가 담당하며, 가능하면 `server.py`의 SQLite API를 사용하고 없으면 `localStorage`로 폴백한다.
- 곡 식별은 점진적으로 `number` 중심에서 일반 `id` 중심 구조로 옮기고 있다.

## 이번까지 구현된 핵심 기능

### 프레젠테이션
- `hymns.json`에서 곡을 불러와 프레젠테이션을 렌더링한다.
- 46장은 `demo-hymn-data.js`의 데모 악보를 합쳐서 표시한다.
- 저장본이 있으면 기본 데이터보다 우선해서 불러온다.

### 가사/악보 편집기
- 편집 모드 토글 (`E`)
- 음표 생성, 박자 변경, 삭제
- 기존 음표 클릭/호버/드래그 기반 음정 변경
- 가사 직접 편집 (`contenteditable`)
- 가사 줄 분할/병합 (`Enter`, `Backspace`, `Delete`)
- `<후렴>` 마커 기반 후렴 슬라이드 전환
- 현재 절/후렴 슬라이드 추가/삭제
- 가사 수정 시 오선지 길이와 음표 위치 즉시 재계산
- 초기 슬라이드 진입 시에도 폰트 로딩 후 레이아웃 재계산
- 조표 편집
- 기존 음표 인라인 메뉴 기반 길이/점음표 편집
- 빈 오선지 드래그 + `Ctrl`/`Shift` 클릭 기반 연결선 선택
- Undo/Redo (`Ctrl+Z`, `Ctrl+Y`, `Ctrl+Shift+Z`)
- JSON 복사/다운로드
- 저장, 저장본 삭제, JSON import

## 현재 편집 규칙
- 일반 편집 모드:
  가사 위 오선지 영역을 호버하면 음표 프리뷰가 보인다.
- 일반 편집 모드:
  빈 자리 클릭으로 새 음표를 만든다.
- 일반 편집 모드:
  기존 음표 위에서 다른 높이를 가리킨 뒤 클릭하면 음정이 바뀐다.
- 일반 편집 모드:
  기존 음표를 드래그하면 음정을 바꿀 수 있다.
- 일반 편집 모드:
  더블클릭하면 해당 음표를 삭제한다.
- 조표 편집:
  음자리표 오른쪽 조표 영역 호버/클릭으로 조표를 추가한다.
- 조표 편집:
  `Ctrl`과 함께 조표 영역을 다루면 샵 기준으로 추가한다.
- 조표 편집:
  조표 영역 더블클릭으로 마지막 조표를 제거한다.
- 기존 음표 인라인 메뉴:
  `길이 변경`, `점 추가`/`점 제거`, `삭제`를 바로 실행한다.
- 연결선 선택:
  기존 8분음표/16분음표만 선택 가능하다.
- 연결선 선택:
  빈 오선지 드래그나 `Ctrl`/`Shift`+클릭으로 같은 줄의 여러 음표를 선택할 수 있다.
- 연결선 선택:
  선택은 한 줄씩만 유지한다. 다른 줄을 고르면 이전 선택은 정리된다.
- 연결선 선택:
  선택한 음표 근처 인라인 메뉴에서 `연결선 적용`, `연결선 해제`, `선택 취소`를 바로 실행한다.
- 연결선 선택:
  빈 공간 클릭 또는 `Esc`로 현재 선택을 해제한다.
- 연결선 선택:
  선택한 음표 사이에 다른 음표가 끼어 있으면 한 번에 같은 연결선 그룹으로 묶지 않는다.

## 저장/불러오기 규칙
- 현재 곡 전체 데이터를 export 대상으로 사용한다.
- export에는 가사 수정 결과와 `notes`가 함께 포함된다.
- `storage.js`는 곡 `id` 기준으로 저장본을 관리한다.
- 프레젠테이션 앱과 편집기는 같은 저장본을 공유한다.

### 2026-04-13 NWC → DB 악보 변환 파이프라인 구축
- `sample/nwc찬송/` 폴더의 NWC 1.70 바이너리 파일(557개)에서 멜로디를 추출하는 파이프라인을 만들었다.
- `nwc_bridge.mjs`: Node.js에서 zz85/nwc-viewer 파서를 호출해 NWC 파일을 파싱하고, 멜로디(pitch+duration) JSON을 stdout으로 출력하는 브릿지 스크립트.
- `nwc_to_hymns.py`: Python에서 nwc_bridge를 subprocess로 호출하고, 추출된 멜로디를 가사 슬라이드/줄 구조에 1:1 매핑하여 `notes` 필드를 추가한다.
- `nwc_batch_convert.py`: 557개 NWC 파일을 일괄 변환하여 DB에 저장하는 스크립트. 555곡 성공.
- `nwc_parser.py`: 초기 Python 직접 파서 (분석용, 최종 파이프라인에서는 미사용).
- NWC V170 NoteCM 구조: `data1(12bytes) + count(2bytes) + children(count × 8bytes)`. 마지막 child가 소프라노(멜로디).
- NWC pitch는 표준 음이름(A4, B4 등)으로 추출하며, v2 pitch 변환은 적용하지 않는다.

### 2026-04-13 후반 수정
- 음표 헤드의 실제 렌더링 중심과 편집기 상호작용 좌표가 어긋나던 문제를 정리했다.
- 선택 원, 클릭 히트테스트, 음표 메뉴 위치는 `getNoteVisualPosition()` 기준으로 맞췄다.
- 새 음표 추가와 기존 음표 드래그의 pitch 판정은 다시 오선지 기준으로 분리했다.
- pitch 라벨 체계를 한 단계 낮춰 재정의했다.
  - 예: `D5 -> C5`, `C5 -> B4`, `B5 -> A5`
- `notes.js`의 `pitchMap`을 새 표기 체계에 맞게 갱신했다.
- `storage.js`, `editor.js`, `app.js`에 `pitchLabelVersion = 2` 마이그레이션을 넣어서 기존 저장본/불러오기 데이터가 한 번만 자동 변환되도록 했다.
- 보조선 규칙을 새 pitch 표기 체계에 맞게 조정했다.
  - `G5`, `B5` 상단 불필요 보조선 제거
  - `C4` 중앙 보조선 표시
  - 하단(`C4` 이하) 보조선은 반 칸 아래 보정
  - 상단(`A5` 이상) 보조선은 한 칸 위로 재조정
- 조표 변경 시 화면 상단 메타 정보도 즉시 갱신되도록 수정했다.
  - 찬송가 정보 줄의 `음조`
  - 현재 슬라이드 정보 줄의 `음조 · 박자`
- 연결선 드래그 선택의 `x` 좌표계가 SVG 내부 좌표와 섞이던 문제를 수정했다.
  - 드래그 박스 `x` 판정은 `lineEl._layout.positions[charIndex]` 기준으로 통일
- 연결선 후보 선택 규칙을 더 엄격하게 바꿨다.
  - `8분음표`와 `16분음표`는 함께 연결선 선택 불가
  - 인접하지 않은 음표를 함께 선택하려 하면 선택 자체를 막고 `"인접하지 않은 음표들이 선택되었습니다."` 경고 출력
  - 위 검증은 드래그 박스 선택과 `Ctrl`/`Shift` 클릭 추가 선택 모두에 적용

## 현재 연결선 규칙 추가 메모
- 연결선 선택은 같은 줄에서만 유지된다.
- 연결선 선택은 같은 기본 길이의 음표끼리만 가능하다.
  - `8 + 8`, `16 + 16` 가능

- 선택 구간 사이에 다른 음표가 끼어 있으면 후보 선택 자체를 막는다.
- 잘못된 연결선 후보는 “적용 단계”가 아니라 “선택 단계”에서 바로 막는다.

### NWC → DB 변환 규칙
1. **Melisma 정렬**: NWC는 슬러/타이 끝·`LyricSyllable=Never` 플래그 음표에 가사를 할당하지 않아 `lyrics` 배열이 `melody`보다 짧음. `melisma_positions`에 `' -'` 토큰을 삽입해 1:1 정렬.
2. **괄호 제거**: `(거 닐 때)`, `(후렴)`, `(123)` 등 악보 상 반복 표기. 문자 단위로 파싱해 괄호 안 내용만 제거(괄호 밖 음절은 보존: `(123)내` → `내`). 해당 음표도 같이 drop.
3. **Accidental**: NWC `getAccidental()` 결과를 `'sharp'|'flat'|'natural'` 문자열로 JSON에 저장.
4. **Beam**: `BeamBeg/Mid/End` 플래그를 연속 그룹으로 묶어 `beamGroup` 정수 ID 부여.
5. **후렴 분리**: DB에 `chorus` 필드가 있을 때만. 1순위=NWC 가사줄 길이 차(`len(line0) > len(line1)`이면 초과분이 후렴), 2순위=DB 후렴 텍스트로 검색.
6. **절별 토큰**: 1절은 `lyrics_lines[0]`의 verse 부분, N절(N≥2)은 `lyrics_lines[N-1]`. 부족 시 line0의 뒤쪽으로 pad.
7. **Key/Time**: NWC 파서의 값이 부정확한 경우가 있어 원본 DB 값을 그대로 보존(교체하지 않음).
8. **토큰 내 하이픈**: `-죄`처럼 음절에 붙은 하이픈은 NWC의 시각 장식이므로 제거하고 음절만 추출. 단독 `-` 토큰만 진짜 멜리즈마 표기로 취급.

## 2026-04-14 편집기 수동 편집 보조 기능

수동으로 악보를 다듬을 때 어긋남을 빨리 식별하고, 음표 슬롯을 한 칸 단위로 옮길 수 있도록 편집기를 보강했다.

### 정합성 시각화 (Dangling 표시)
- 가사 글자에 매칭되는 음표가 없거나, 가사 길이를 넘어선 음표가 있으면 빨간색(`#d8324c`)으로 강조.
- 가사 텍스트는 contenteditable이라 직접 색상을 입히면 커서/입력에 간섭이 생기므로, `.editor-line-text-wrap` > `.editor-line-text-overlay` 구조로 별도 오버레이를 띄움. 오버레이는 `pointer-events:none; color:transparent`이고 dangling char만 빨강 span으로 렌더 → 시각적으로 글자가 빨갛게 보이지만 입력은 그대로 통과.
- Dangling 음표는 SVG의 `fullWidth`를 확장해 가사 끝 오른쪽에 빨간색으로 추가 그림.
- 입력으로 가사 길이가 줄어들 때 `syncNotesToCurrentText`가 잘려나가는 음표를 배열 끝으로 밀어 dangling으로 보존하도록 수정. (이전엔 잘림 → 편집 중 dangling이 사라지는 버그)

### 인라인 메뉴 확장
- 닫기(`✕`) 버튼 제거. 메뉴는 다른 곳 클릭/다른 음표 선택으로 닫음.
- `<` / `>` 버튼 추가: 선택 음표부터 줄 끝까지의 슬롯을 한 칸 당기거나 밀기. `<`는 선택 음표가 줄 첫 칸이거나 직전 슬롯에 음표가 있으면 비활성화. `>`는 항상 가능 (dangling 생성 허용).
- 임시표 비활성화 규칙: 조표가 해당 음을 ♭/♯ 처리하면 같은 방향 임시표 비활성화, 조표 영향이 없는 음에서는 ♮ 비활성화. 단 이미 적용된 임시표는 해제 가능하도록 비활성화하지 않음.
- 음 이름은 cycle of fifths(`B,E,A,D,G,C,F` 플랫 / `F,C,G,D,A,E,B` 샵) 순서로 조표 카운트만큼 잘라 비교.
- 클릭 핸들러에서 `disabled` 검사 추가해 비활성 버튼은 무시.
