# LyricDisplay 개발 기획서 v3

> 이 문서는 초기 기획과 제품 방향을 정리한 문서이며, 최신 구현 상태와 변경 이력은 `개발일지.md`를 기준으로 한다.

## 1. 프로젝트 개요

### 1.1 프로젝트 목적
교회 예배를 위한 찬양 가사 프레젠테이션 웹 애플리케이션 개발. 찬송가와 CCM 가사를 효율적으로 관리하고, **텍스트 콘텐츠에 집중할 수 있는 미니멀한 프레젠테이션**을 제공하는 시스템.

### 1.2 디자인 철학
> **"텍스트와 콘텐츠만이 주인공"**
>
> 기존 프레젠테이션 도구의 과도한 장식과 효과를 배제하고, 가사 텍스트 자체에 집중할 수 있는 미니멀리즘 디자인을 지향한다. 세련된 타이포그래피와 고정된 레이아웃으로 일관된 시각적 경험을 제공한다.

### 1.3 주요 목표
- 찬양 순서의 직관적인 작성 및 관리
- 찬송가 한글-영문 이중 표시 지원
- CCM 이미지 슬라이드 및 텍스트 가사 관리
- **미니멀한 풀스크린 프레젠테이션 (Vanilla JS)**
- 실시간 검색 및 편집 기능
- 미디어(이미지, 동영상) 통합 관리
- **[NEW] 악보(Notes) 표시 기능 - 한글 가사 위에 오선지와 음표 렌더링**

### 1.4 타겟 사용자
- 교회 예배 담당자
- 찬양팀 리더
- 미디어 팀 운영자
- **[NEW] 성가대 지휘자 / 반주자 (악보 기능 활용)**

## 2. v3 신규 기능: 악보 (Notes)

### 2.1 기능 개요

한글 가사 각 줄 위에 오선지와 음표를 표시하여, 찬양 가사와 함께 멜로디를 시각화하는 기능.

#### 핵심 특징
| 항목 | 내용 |
|------|------|
| 렌더링 방식 | 경량 SVG 직접 구현 (VexFlow 미사용) |
| 음표 폰트 | Bravura (SMuFL 표준, SIL Open Font License) |
| 오선지 색상 | CSS 변수 `--color-gray` (기본값: `#999`) |
| 음표 색상 | 검정색 (`#000`) |
| 강조 색상 | CSS 변수 `--color-accent` (기본값: `#FF6B35`) |
| 정렬 | 각 글자와 음표의 가로 중앙 정렬 |
| 조건부 표시 | 음표 데이터가 없으면 오선지 미표시 |
| 반응형 | 768px 이하에서 0.8배 축소 |

### 2.3 음높이(Pitch) 매핑

```
오선지 (높은음자리표):
─────────── F5 (다섯째 줄)
─────────── D5 (넷째 줄)
─────────── B4 (셋째 줄, 가운데)
─────────── G4 (둘째 줄)
─────────── E4 (첫째 줄)
- - - - - - C4 (아래 첫째 덧줄)
```

| 음높이 | 오선지 위치 | 설명 |
|--------|-------------|------|
| C4 | 아래 첫째 덧줄 | 가온다 |
| D4 | 첫째줄 아래 | |
| E4 | 첫째 줄 | |
| F4 | 첫째줄~둘째줄 사이 | |
| G4 | 둘째 줄 | |
| A4 | 둘째줄~셋째줄 사이 | |
| B4 | 셋째 줄 (가운데) | |
| C5 | 셋째줄~넷째줄 사이 | |
| D5 | 넷째 줄 | |
| E5 | 넷째줄~다섯째줄 사이 | |
| F5 | 다섯째 줄 | |
| G5 | 다섯째줄 위 | |
| A5 | 위 첫째 덧줄 | |

### 2.4 박자(Duration) 매핑

| 코드 | 음표 | 모양 |
|------|------|------|
| `w` | 온음표 | ○ (속 빈 타원, 기둥 없음) |
| `h` | 2분음표 | ○─ (속 빈 타원 + 기둥) |
| `q` | 4분음표 | ●─ (속 찬 타원 + 기둥) |
| `8` | 8분음표 | ●─꼬리1 |
| `16` | 16분음표 | ●─꼬리2 |

#### 기본 박자 (timeSignature 기준)
```javascript
getDefaultDuration(timeSignature) {
    const denominator = parseInt(timeSignature.split('/')[1]);
    switch (denominator) {
        case 2: return 'h';   // 2분박
        case 4: return 'q';   // 4분박
        case 8: return '8';   // 8분박
        default: return 'q';
    }
}
```

### 2.8 조표 (Key Signature) 기능

#### 개요
음자리표 오른쪽에 조표(샵/플랫)를 표시하여 해당 곡의 조성을 나타냄.

#### 조표 데이터 형식
```javascript
// hymn JSON의 key 필드
"key": "4b"     // 4플랫 (Ab장조/F단조)
"key": "3#"     // 3샵 (A장조/F#단조)
"key": "Bb"     // 조 이름으로도 지정 가능
```

#### 조표 파싱
| 입력 | 타입 | 개수 |
|------|------|------|
| `"4b"` | flat | 4 |
| `"3#"` | sharp | 3 |
| `"Bb"` | flat | 2 |
| `"D"` | sharp | 2 |
| `"C"` or `"F"` | none | 0 |

#### 조표 위치 (오선지 기준, 0=F5 top line, 4=E4 bottom line)
```
플랫 순서: Bb → Eb → Ab → Db → Gb → Cb → Fb
        (1.5) (0)  (2) (0.5)(2.5) (1)  (3)

샵 순서:  F# → C# → G# → D# → A# → E# → B#
         (3)  (1) (2.5)(0.5) (2)  (0) (1.5)
```

#### SVG 렌더링
- 조표는 음자리표 오른쪽, 음표 왼쪽에 배치
- 조표 간격: 10px
- 조표 너비에 따라 오선지가 왼쪽으로 연장됨
- 색상: 음자리표와 동일한 `--color-gray`

## 3. 편집 모드 (향후 구현)

### 3.1 개요

사용자가 오선지 위에서 직접 음표를 입력/편집할 수 있는 인터랙티브 편집 모드.

### 3.2 UI/UX 명세

#### 호버 미리보기
```
┌────────────────────────────────────────┐
│  ─────●─────────────────────────────   │ ← 마우스 위치에 회색 음표 미리보기
│  ─────────────────────────────────────  │
│  ─────────────────────────────────────  │
│  ─────────────────────────────────────  │
│  ─────────────────────────────────────  │
├────────────────────────────────────────┤
│       찬  양  하  라  복  되  신        │
└────────────────────────────────────────┘
          ↑
     마우스 호버 위치
```

- 마우스가 오선지 위를 호버링하면:
  - 해당 X 좌표에 가장 가까운 한글 글자 식별
  - 해당 Y 좌표의 음높이 계산
  - 기본 박자(`timeSignature` 기준)의 회색 음표 미리보기 표시

#### 클릭 입력
| 입력 | 동작 |
|------|------|
| 왼쪽 클릭 | 음표 생성 (기본 박자) 또는 박자 증가 |
| Ctrl + 왼쪽 클릭 | 박자 감소 |
| 더블 클릭 | 음표 삭제 |

#### 박자 순환
```
[클릭] 16분 → 8분 → 4분 → 2분 → 온음표 → 16분 ...
[Ctrl+클릭] 온음표 → 2분 → 4분 → 8분 → 16분 → 온음표 ...
```

#### 드래그 (음높이 변경)
```
┌────────────────────────────────────────┐
│  ───────────────────────●←─────────────│ ← 드래그로 음높이 변경
│  ─────────────────────────────────────  │
│  ─────────────●─────────────────────   │ ← 원래 위치
│  ─────────────────────────────────────  │
│  ─────────────────────────────────────  │
└────────────────────────────────────────┘
```

- 확정된 검은색 음표를 드래그하여 음높이(pitch) 변경
- 드래그 중 음표는 반투명 표시
- 놓으면 새 음높이로 확정

### 3.3 편집 모드 데이터 흐름

```
1. 사용자 호버
   └─> measureMousePosition(event)
       └─> findNearestChar(x)
       └─> calculatePitch(y)
       └─> renderPreview(charIndex, pitch, defaultDuration)

2. 사용자 클릭
   └─> handleClick(event)
       └─> if (existingNote) { cycleDuration(+1/-1) }
       └─> else { createNote(charIndex, pitch, defaultDuration) }
       └─> updateNotesData()
       └─> renderNotes()

3. 사용자 드래그
   └─> handleDragStart(event)
       └─> selectNote(charIndex)
   └─> handleDragMove(event)
       └─> calculatePitch(y)
       └─> renderDragPreview()
   └─> handleDragEnd(event)
       └─> updateNotePitch(charIndex, newPitch)
       └─> renderNotes()
```

### 3.4 조표 편집

#### 조표 입력 영역
- 음자리표 오른쪽 근처 (조표 영역)에서 마우스 호버 시 조표 미리보기 표시

#### 호버 미리보기
```
┌────────────────────────────────────────┐
│  𝄞 ♭ ←──────────────────────────────   │ ← 마우스 위치에 회색 플랫 미리보기
│  ─────────────────────────────────────  │
│  ─────────────────────────────────────  │
│  ─────────────────────────────────────  │
│  ─────────────────────────────────────  │
└────────────────────────────────────────┘
    ↑
 마우스 호버 위치 (Y좌표로 음높이 결정)
```

#### 입력 방식
| 입력 | 동작 |
|------|------|
| 호버 (조표 영역) | 플랫(♭) 미리보기 표시 |
| Ctrl + 호버 | 샵(♯) 미리보기 표시 |
| 왼쪽 클릭 | 조표 추가/확정 |
| 더블 클릭 | 조표 삭제 |

#### 조표 순서 규칙
- 플랫: Bb → Eb → Ab → Db → Gb → Cb → Fb (순서대로만 추가 가능)
- 샵: F# → C# → G# → D# → A# → E# → B# (순서대로만 추가 가능)
- 클릭 시 다음 순서의 조표가 자동 추가됨

### 3.5 점음표 편집

#### 점음표 토글 모드
- `D` 키를 눌러 점음표 편집 모드 진입/종료

#### 동작 방식
```
┌────────────────────────────────────────┐
│  [D 모드 활성화]                         │
│  𝄞 ─────────────────────────────────── │
│       ♩   ♩̣  ←── 기존 음표에 호버       │
│       찬  양  하  라                    │
└────────────────────────────────────────┘
           ↑
    회색 점 미리보기 (호버 시)
```

#### 입력 방식
| 입력 | 동작 |
|------|------|
| `D` 키 | 점음표 편집 모드 토글 |
| 호버 (기존 음표) | 회색 점 미리보기 표시 |
| 왼쪽 클릭 | 점 추가/제거 토글 |

#### 점 위치 규칙
- 음표 머리 오른쪽에 배치 (0.9 * lineSpacing 오프셋)
- 음표와 동일한 Y 좌표에 점 배치

### 3.6 연결선(Beam) 편집

#### 연결선 기능
- 8분음표, 16분음표 등을 가로로 연결하는 선
- `beamGroup` 속성으로 같은 그룹의 음표들을 연결

#### 데이터 형식
```javascript
{pitch: "A4", duration: "16", beamGroup: 1}, // 연결선 그룹 1
{pitch: "B4", duration: "16", beamGroup: 1}, // 연결선 그룹 1
{pitch: "A4", duration: "8", beamGroup: 2},  // 연결선 그룹 2
{pitch: "B4", duration: "8", beamGroup: 2},  // 연결선 그룹 2
```

#### 편집 모드 동작 방식
```
┌────────────────────────────────────────┐
│  𝄞 ─────────────────────────────────── │
│      [♪]  [♪]  [♪]  ←── 음표 다중 선택  │
│       찬   양   하   라                 │
│              ↓                          │
│          [연결선 메뉴]                   │
└────────────────────────────────────────┘
```

#### 입력 방식
| 입력 | 동작 |
|------|------|
| 드래그 선택 | 여러 음표 영역 선택 |
| Ctrl + 클릭 | 음표 다중 선택 추가/제거 |
| 메뉴 클릭 | 선택한 음표들에 연결선 적용 |
| 연결선 더블클릭 | 연결선 삭제 |

#### 연결선 렌더링 규칙
- 8분음표: 1개의 연결선 (두께 5)
- 16분음표: 2개의 연결선
  - 첫 번째 연결선: 두께 5 (8분음표와 동일한 Y 위치)
  - 두 번째 연결선: 두께 1.5 (첫 번째 연결선 아래에 배치)
- 기둥 방향: 그룹 음표들의 평균 음높이로 결정 (B4 기준)
- 연결선 위치: 모든 기둥 끝을 연결
- 연결선 간격: 4px (첫 번째와 두 번째 연결선 사이)
